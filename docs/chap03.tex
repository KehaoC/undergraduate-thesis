\chapter{Taste 系统总体设计}
\label{cha:design}

\section{原型系统目标}
\label{sec:arch_overview}

本文实现的 Taste 原型系统围绕一个明确目标展开：将分散在 GitHub 仓库中的 Agent Skill 统一整理为可计算的数据资产，在离线阶段完成语义组织与质量评估，在在线阶段以三维语义地形图的方式向用户展示，并支持按任务描述返回最合适的 Skill 子集。与“先建一个完整平台，再等待数据与算法补齐”的做法不同，本文采用“\textbf{数据集构建 $\rightarrow$ 离线排序 $\rightarrow$ 在线 Demo}”的最小闭环方案，把论文中需要论证的关键问题直接落到可复现的代码路径上。

这一设计有三个直接好处。第一，所有实验依赖都固定在版本化的 JSON 数据集与离线产物文件上，便于后续反复复现实验；第二，排序算法与交互前端完全解耦，算法每次重算后只需替换产物文件即可更新 Demo；第三，在线部分不承担昂贵的全量重计算，只负责读取预计算结果与执行轻量级查询路由，因此能够以很低的工程复杂度完成“算法价值 $\rightarrow$ 用户体验”的转化。

\section{SkillRank 计算链路设计}
\label{sec:skillrank_module}

本文最终采用的 SkillRank 计算链路如图 \ref{fig:skillrank_pipeline} 所示。与最初“全局图上直接 PageRank”的直觉实现不同，最终方案明确遵循第二章提出的\textbf{条件排序}定义：先用 Description embedding 把 Skill 划分为若干语义竞争邻域，再在每个邻域内部计算内容质量分，最后把这些分值与二维坐标一同导出为可视化产物。

  \node[stage] (S1) at (0,0) {Description / Content\\Embedding};

  \node[note] at (3.2,-1.6) {MiniBatch KMeans\\簇数按数据规模启发式确定\\定义竞争邻域};
  \node[note] at (6.4,-1.6) {先计算 Content 原型中心度\\再在簇内构图运行 PageRank\\融合为最终 SkillRank};
  \node[note] at (9.6,-1.6) {PCA $\rightarrow (x,y)$\\SkillRank $\rightarrow z$\\导出 \texttt{terrain.json}};
\end{tikzpicture}
\caption{SkillRank 离线计算链路。系统先在 Description 空间划定竞争邻域，再在每个邻域内部融合 Content 原型先验与 PageRank 风格的图传播分。}
\label{fig:skillrank_pipeline}
\end{figure}

这里有两个关键设计决策。其一，聚类使用 Description 而非 Content。原因在于，Description 表达的是“Skill 是做什么的”，更适合用于定义比较范围；若直接在 Content 空间聚类，容易把风格相似但功能不同的 Skill 混到同一簇。其二，最终分值不是单独依赖图传播，而是采用\textbf{Content 原型中心度 + PageRank 图传播分}的融合形式。前者衡量某个 Skill 是否接近该语义簇内的主流内容原型，后者衡量它是否处在由近邻相互“引用/模仿”形成的结构中心。二者组合后，能够在本文的数据尺度下比单纯的图分或单纯的中心度都更稳定。

\section{在线查询路由设计}
\label{sec:query_routing}

Taste Demo 的在线查询流程如图 \ref{fig:query_flow} 所示。为了保证演示阶段的稳定性，在线部分不再实时计算句向量，而是采用轻量级的词法召回与离线分数重排：前端提交任务描述后，后端先用 TF-IDF 在 Description 文本上召回候选，再在候选集中按预计算的 SkillRank 分值重排，返回前 $K$ 个结果。这一设计把“实时语义编码”的复杂度留在离线实验中，把“在线响应稳定性”留给 Demo 系统。

这一流程的关键点在于“\textbf{候选召回与质量排序分离}”：TF-IDF 负责回答“哪些 Skill 在字面语义上最像当前任务”，SkillRank 负责回答“在这些候选里哪些更值得优先展示”。这种两阶段结构能够保持在线逻辑简单，同时继续保留离线质量评估的价值。

\section{本章小结}
\label{sec:chap3_summary}

本章从系统设计角度明确了 Taste 原型的实现边界：以 GitHub Skill 数据集为底层资产，以离线聚类与 SkillRank 计算作为核心引擎，以 JSON/NPY 产物作为在线 Demo 的唯一输入，并以 Plotly.js 三维语义地形图完成可视化呈现。这样的设计既服务于论文实验的可复现性，也保证了系统演示能够直接建立在真实实验结果之上。第四章将进一步说明这些设计在代码中的具体落地方式。
