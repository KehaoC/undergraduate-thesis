\chapter{绪论}
\label{cha:introduction}

\section{选题背景与意义}
\label{sec:background}

大语言模型（Large Language Model，LLM）驱动的智能体（Agent）技术正在经历一次前所未有的能力跃迁。以 Claude Code、Codex、Manus 为代表的闭源 Agent，以及 OpenClaw、AutoGPT 等开源 Agent，正在通过接入各类工具将其能力边界从纯语言生成扩展至可操作的真实世界任务。这些 Agent 的基座模型大多针对代码任务进行了深度训练，在推理时能够将自然语言需求翻译为终端命令、API 调用和文件操作序列，而命令行界面本身就是操作计算机的原语——这意味着，一个足够能干的 Agent 在理论上可以完成人类能够通过计算机完成的一切任务。OpenClaw 在 GitHub 上积累的 Star 数已超过 React 和 Linux 等知名开源项目，这一数字本身折射出整个技术社区对 Agent 范式的强烈期待。

然而，Agent 的实际表现与这种期待之间仍存在显著落差，而这一落差的根源，在很大程度上与 Skill 的使用方式有关。在主流的 Agent 框架中，Skill（也被称为 Plugin、Tool Spec 或 System Prompt 片段）是 Agent 获取领域专业能力的核心途径。每一个 Skill 本质上是一个精心设计的上下文注入单元——它在 Agent 处理任务之前被插入到系统提示中，为 Agent 提供特定领域的知识、操作规范或行为约束。一个熟练使用 Git 的 Agent 和一个不使用任何 Skill 的 Agent，在代码版本管理任务上的表现可能相差悬殊，而这一差距完全来自于一个几百字的 Skill 文本是否被正确地注入了上下文。

当前公开的 Skill 生态已经在 GitHub 仓库、社区索引站与个人技能仓中积累了数以万计的可复用 Skill，覆盖浏览器操作、代码生成、文档写作、数据分析、API 集成等数十个技术领域。这一规模的 Skill 库在提供了丰富选择的同时，也制造了一个新的问题：大量的 Skill 反而使 Agent 的能力受限。这听起来似乎矛盾，但其根源在于 Agent 使用 Skill 的方式存在根本性缺陷。

\textbf{第一个问题是本地存储模式导致的意图匹配准确率衰减。}在现有的 Agent 框架中，用户需要将 Skill 下载到本地，Agent 在每次处理任务时从本地 Skill 集合中选择最相关的若干条注入上下文。这一过程依赖 Agent 自身的意图判断能力——Agent 必须通过比较任务描述与每个 Skill 的 Description 来决定哪些 Skill 与当前任务相关。研究表明\cite{liu2024lostinthemiddle}，当上下文中的候选项数量超过一定阈值时，LLM 对候选项的区分能力会急剧下降。具体而言，在 Skill 选择任务中，当本地 Skill 数量少于 20 个时，Agent 的选择准确率通常可以维持在 90\% 以上；当数量超过 30 个时，准确率开始出现统计显著的下降；当数量达到 200 个时，准确率已跌至约 20\%，与随机猜测相差无几。在这种状态下，Skill 再多也无法转化为 Agent 能力的提升——冗余的 Skill 不仅无法被准确选中，还会作为噪声干扰 Agent 对真正有用 Skill 的识别。

\textbf{第二个问题是 Skill 的发现与筛选完全依赖用户的技术认知。}当前 Skill 平台的使用流程与二十年前的软件下载站惊人地相似：用户通过关键词搜索或浏览分类列表找到候选 Skill，下载并手动添加到本地 Skill 集合，再根据个人经验决定是否保留。这一流程对用户有隐性的技术门槛要求：用户必须了解自己的 Agent 框架支持何种格式的 Skill、必须能够判断一个 Skill 的 Content 是否确实对其常用任务有帮助、必须定期维护本地 Skill 集合以去除过时或低质的条目。对于绝大多数没有技术背景的普通用户而言，这些要求意味着他们的 Agent 能力上限被直接锁定在其个人认知水平之上——不论 Skill 生态有多丰富，他们实际上只能使用其中极小的一部分。

\textbf{第三个问题是 Skill 生态缺乏客观的质量评估体系。}现有公开目录大多以 GitHub Star 数、仓库热度或下载量作为主要排序依据。这一方案的问题不仅在于 Star 数有技术认知门槛（非开发者用户无法贡献），更在于它从根本上忽视了 Skill 价值的情境依赖性。一个面向浏览器自动化的 Skill 积累了三年的 Star，与一个刚发布的代码格式化 Skill 相比，前者的更高排名并不意味着它在代码格式化任务上更好——事实上两者根本不构成竞争关系，将它们放在同一个排行榜上做比较在逻辑上就是错误的。这种全局混排的方式使得高质量的垂直领域 Skill 长期被埋没在大量热门但不相关的条目之中。

上述三个问题相互叠加，在 Skill 生态高速扩张的背景下形成了一道日益加高的墙：Skill 越来越多，但用户能有效利用的比例反而越来越低。解决这一问题，不是优化 Skill 的创作工具或发布流程，而是需要从根本上重新设计 Skill 的组织与发现方式。

正如互联网在网页数量爆炸式增长之初面临的信息检索问题，最终由 Google 通过 PageRank 和系统化的网页建模给出了答案，Agent Skill 生态正在等待一个类似的解法。本文的工作正是对这一方向的探索：设计并实现 Taste 系统原型及其核心算法 SkillRank，对 Skill 世界进行系统性建模，并以离线实验加在线 Demo 的方式验证这种建模是否能够真正支撑 Skill 发现与筛选。

\section{国内外研究现状}
\label{sec:related_work}

\subsection{Agent 与 Skill 生态的发展}

Agent 的工具使用能力研究在近年来得到了广泛关注。Yao 等人提出的 ReAct 框架\cite{yao2023react}将语言推理与工具调用动作交织融合，使 Agent 能够在推理过程中动态决定何时调用工具，并将工具的返回结果融入后续推理链，这是 Agent 工具使用研究的奠基性工作之一。Schick 等人的 Toolformer\cite{schick2023toolformer}展示了如何通过自监督方式训练语言模型自主学习在何处插入工具调用，无需昂贵的人工标注。Patil 等人的 Gorilla\cite{patil2023gorilla}专门针对 API 调用场景，构建了包含大量 API 文档的训练集，显著提升了模型在真实 API 使用场景下的准确率。Qin 等人的 ToolLLM\cite{qin2024toolllm}更进一步，构建了覆盖 16,000 余个真实 API 的基准测试集，系统性地评估了不同 LLM 在大规模工具环境下的使用能力。

在 Skill 管理与组合层面，Shen 等人的 HuggingGPT\cite{shen2023hugginggpt}提出了一种以 ChatGPT 作为控制器、调用 Hugging Face 上各类 AI 模型作为工具的系统，其核心洞察与本文高度相关：任务规划和工具分配是大规模工具使用的核心瓶颈，而不是单个工具的质量。Liang 等人的 TaskMatrix.AI\cite{liang2024taskmatrix}提出了一个连接基础模型与数百万 API 的统一框架，通过 API 文档生成、语义匹配和结果验证形成闭环，但该工作侧重于 API 的功能性正确调用，对 API 或 Skill 本身的质量评估未作深入探讨。

在 Skill 发现与评估领域，目前的学术研究明显滞后于工程实践。现有公开目录和社区索引站在设计上都更像是内容聚合平台而非智能发现系统，其排序通常依赖 GitHub Star、下载量或人工标签，这些指标均存在明显的马太效应：历史较长、曝光度高的 Skill 持续积累高分，而新发布的高质量 Skill 在起步阶段几乎不可见。换言之，现有系统更擅长“展示已有热度”，而不擅长“识别结构上更值得优先注入的 Skill”。

\subsection{图结构价值传播：PageRank 及其变体}

Brin 和 Page 在 1998 年提出的 PageRank 算法\cite{brin1998anatomy}是图结构价值传播领域的奠基性工作。PageRank 的核心思想是将网页的重要性定义为一种递归关系：一个被重要网页所链接的网页，自身也是重要的。这一递归定义通过幂迭代法在有向链接图上求解，其稳态分布对应随机游走者在图上的平稳分布。Page 等人的后续工作\cite{page1999pagerank}进一步完善了算法的理论框架，证明了在满足图的强连通性（通过引入阻尼因子实现）条件下算法的收敛性保证。

PageRank 在学术引用分析、社交网络影响力评估等领域得到了广泛应用，但将其直接迁移至 Skill 价值评估面临一个根本性障碍：Skill 生态中不存在类似超链接的显式有向关系。本文提出以 Content 语义相似度替代超链接作为图的边，这一替换在语义上是合理的——若两个 Skill 的内容高度相似，则它们在解法层面存在某种亲缘或派生关系，而这种亲缘关系正是价值传播的语义基础。

\subsection{信息检索与语义排序基础}

传统信息检索领域的排序方法为本文提供了重要的方法论参考。Robertson 和 Zaragoza 系统总结的 BM25 框架\cite{robertson2009bm25}通过词频归一化和文档长度惩罚实现了对 TF-IDF 的实质性改进，至今仍是许多生产级检索系统的标准基线。Liu 的综述\cite{liu2009learningtorank}梳理了从 Pointwise 到 Pairwise 再到 Listwise 的学习排序方法演进，指出排序学习的核心挑战在于如何设计能够捕捉相对顺序而非绝对分值的损失函数。

然而，上述方法都预设了一个关键前提：存在一个明确的查询（Query），排序的目标是找到与查询最相关的文档。Skill 排序问题的特殊性在于，排序的依据不是查询相关性，而是 Skill 本身的内在质量——这是一个无监督的绝对质量评估问题，与传统检索的相对相关性评估存在本质区别。Reimers 和 Gurevych 提出的 Sentence-BERT\cite{reimers2019sentencebert}为大规模语义相似度计算提供了高效工具，其孪生网络结构使得在推理时无需对每个文本对单独运行 BERT 前向传播，将语义检索的时间复杂度从 $O(n^2)$ 降至 $O(n)$（建索引阶段）加 $O(1)$（单次查询阶段），这对 Taste 系统中大规模 Skill embedding 的工程实现至关重要。

\subsection{大规模向量检索}

随着 embedding 技术的普及，在百万至十亿级向量集合中执行高效的最近邻搜索成为工程实践中的核心挑战。Johnson 等人开发的 FAISS\cite{johnson2021faiss}通过倒排文件索引（IVF）、乘积量化（PQ）和 HNSW 等多种索引结构，在精度与速度之间提供了灵活的权衡选项，成为学术界和工业界最广泛使用的向量检索库。本文在 SkillRank 的图构建阶段大量依赖 FAISS，将 K-NN 图构建复杂度从理论上的 $O(n^2 d)$ 降至实践中的 $O(n\log n)$，是实现系统线性可扩展性的关键工程决策。

\subsection{自动化实验优化}

Lu 等人的 AI Scientist\cite{lu2024aiscientist}是近年来自动化科研领域的代表性工作，展示了 Agent 在一定约束下自主提出假设、设计实验、分析结果并迭代改进的能力。本文借鉴其"单变量控制 + 自动迭代"的核心思想，设计了 AutoResearch 风格的超参数调优框架，系统性地优化 SkillRank 算法的关键参数，并将优化过程本身作为可复现的实验结果呈现。

\section{研究内容与目标}
\label{sec:research_content}

本文的核心研究问题是：如何在无需人工质量标注的前提下，对大规模 Agent Skill 进行系统性的价值评估与条件排序，并将其落地为可复现、可演示、可继续迭代的系统原型？

围绕这一问题，本文的研究内容分为三个层次。在理论层面，本文对 Skill 的结构特性进行形式化分析，提出条件排序的问题定义，并建立 Content 亲缘图上的价值传播机制。在算法层面，本文设计了 SkillRank 算法，通过 FAISS 近似最近邻搜索为更大规模数据保留稀疏图构建路径，并通过 AutoResearch 框架对关键超参数进行系统性调优；在当前实际执行的 100 和 1000 两个规模实验上，调优后的 SkillRank 均优于最强非图结构基线。在工程层面，本文完成了 Taste 系统原型的实现，涵盖数据资产组织、离线 SkillRank 计算、在线查询重排和三维可视化前端，交付一个可直接运行的系统原型。

\section{主要工作与创新点}
\label{sec:contributions}

本文的第一个核心贡献是提出了 Skill 价值评估的\textbf{条件排序建模框架}。通过将 Skill 形式化为 $\langle Description, Content \rangle$ 二元结构，本文揭示了需求定位（由 Description 承载）与内容质量（由 Content 承载）是两个相互独立的问题。在此基础上，本文将 Skill 价值评估精确定义为条件排序问题：只有在语义邻域（Description 语义相近的 Skill 集合）内部对 Content 质量进行排序才具有意义，跨语义邻域的比较从根本上没有价值。这一定义纠正了现有平台全局混排的根本逻辑错误。

本文的第二个核心贡献是设计了\textbf{SkillRank 算法}，将 PageRank 的图结构价值传播机制迁移至 Skill Content 亲缘图。SkillRank 以 Content 语义相似度替代超链接，并引入“Content 原型中心度 + 图传播分”的融合评分，在无需任何外部人工标注的前提下，通过结构性信号实现对 Skill 内在质量的客观评估。经 AutoResearch 框架迭代优化后，SkillRank 在 100 和 1000 两个规模实验上分别达到 within-cluster Spearman $\rho=0.0600$ 和 $\rho=0.0562$，均优于最强非图结构基线。

本文的第三个贡献是指出了\textbf{Agent 消费 Skill 与人类消费搜索结果在优化目标上的本质差异}。人类对排序位置的注意力呈指数级衰减，优化 Top-1 准确率是合理的；而 Agent 能够在上下文窗口内均等处理所有注入内容，真正的优化目标是选入窗口的 $K$ 个 Skill 的集合整体质量，而非精确找到排名第一的单个 Skill。这一洞察将排序问题从"单点最优"重新定义为"区域覆盖质量"。

本文的第四个贡献是设计并实现了\textbf{Taste 系统原型}，将上述算法落地为完整可运行的研究型 Demo。Taste 提供从数据资产组织、SkillRank 离线计算、语义路由查询到三维语义地形图可视化的完整链路，并保留实验日志、汇总 JSON、截图与可视化产物，为后续迭代提供直接可复用的工程基础。

\section{论文结构安排}
\label{sec:arrangement}

本文共分为五章。第一章为绪论，介绍研究背景、相关工作、研究目标与主要创新点。第二章建立 Skill 的形式化建模框架，推导条件排序定义和 SkillRank 算法的数学基础，分析可扩展性约束与系统设计目标。第三章介绍 Taste 系统的总体设计，包括系统架构、模块划分、产物组织和接口设计。第四章描述系统的具体实现，包括数据资产组织、SkillRank 算法实现、AutoResearch 调优框架、日志保留机制和三维可视化实现。第五章报告实验结果，对 100 与 1000 两个规模上的排序质量、聚类敏感性和效率表现进行定量评估，并在最后总结全文工作与未来研究方向。
