\chapter{实验评测与总结}
\label{cha:experiment}

\section{实验设计}
\label{sec:exp_design}

本章实验围绕三个问题展开：第一，SkillRank 是否能够在语义邻域内部给出优于非图结构基线的排序；第二，聚类粒度与参数调优是否会显著改变实验结论；第三，在当前真正执行的 100 和 1000 两个规模上，FAISS 近似检索是否已经带来实际的运行时间收益。

\subsection{实验数据与执行边界}

本文以第四章整理得到的 10028 条 GitHub Skill 数据资产为统一输入来源，但本轮定稿阶段只实际运行了 100 和 1000 两个规模的子集。这样的边界是刻意设置的：目标不是在一晚之内盲目堆大规模，而是在可控时间内完整保留两组真实结果、日志与图表。100 规模用于快速观察算法行为与小样本稳定性，1000 规模作为主实验规模；10000 规模代码路径已经具备，但本次不执行。

\subsection{Ground Truth 与评测指标}

本文仍以 GitHub 仓库 Star 数作为弱监督 proxy，而不是把它当作完美真值。具体评测方式为：先在 Description 语义空间内聚类，再在每个聚类内部比较 SkillRank 排序与 Star 排序的 Spearman 秩相关系数，最后按聚类规模加权平均，得到 overall within-cluster Spearman $\rho$。这一指标的含义是：系统是否能够在“同类 Skill”的比较范围内，把社区热度较高或被生态更广泛接受的 Skill 排到更前的位置。

需要说明的是，当前数据集中的多个 Skill 可能来自同一 GitHub 仓库，因此它们会共享同一个 Star 数。这意味着 Star 只能被视为仓库级热度信号，而不是精确的 Skill 级质量标签。本文后续所有分析都在这一限制下进行。

\subsection{对效率结果的理论解读}
\label{sec:efficiency_interpretation}

需要着重说明的是，当前实验在效率维度上呈现的”FAISS 更慢”结论，\textbf{并不与本文的算法设计相矛盾，而恰恰是理论预期的一部分}。

从算法的时间复杂度分析出发（表 \ref{tab:complexity} 已给出），FAISS IVF 的核心优势体现在 $n$ 足够大时。FAISS 的每次查询需要完成聚类中心构建（$O(\sqrt{n} \cdot d)$）、倒排列表索引训练，以及 \texttt{nprobe} 参数控制的探测开销。这些固定启动成本在 $n=100$ 和 $n=1000$ 的规模下仍占总运行时间的主导地位，而精确 K-NN 在此规模下的暴力计算量足够小，反而无需摊销任何启动开销。

然而，随着 $n$ 增大，精确 K-NN 的复杂度以 $O(n^2 d)$ 增长（每次图构建需计算所有节点对的相似度），而 FAISS 的图构建复杂度仅为 $O(n \log n)$——两者的差距将以超线性速度拉开。一个直观的外推：在 $n = 10000$ 时，精确检索的计算量约为 $n=1000$ 时的 $100$ 倍，而 FAISS 的计算量仅增长约 $12$ 倍（$10000 \log 10000 / 1000 \log 1000$）。在 $n = 100000$ 规模时，这一比值将达到约 $3000$ 倍 vs. $160$ 倍。

因此，本文效率实验的正确解读方式是：\textbf{在当前数据规模下，实验证明了 FAISS 近似检索几乎不损失排序质量（$|\rho_{\text{exact}} - \rho_{\text{faiss}}| \leq 0.0011$），同时验证了算法在小规模下的行为符合理论预期}。时间优势的实证闭环需要在 $n \geq 10000$ 规模上完成，这是后续工作的自然延伸，而非当前理论框架的缺陷。本文的核心贡献在于建立了理论上可扩展（$O(n \log n)$）、实践上质量可保持的算法框架，并在已执行规模上完成了对两个性质的分别验证。

这一案例有两个现象值得注意。第一，排名靠前的条目并不一定来自高 Star 仓库，例如 \texttt{openapi-spec-generation} 与 \texttt{moai-lang-typescript} 的仓库 Star 都很低，但在结构上处于该簇内部更中心的位置。第二，末尾条目并非简单对应最低 Star 数，而是通常同时具备“结构中心性弱”和“与簇原型偏离更大”这两个特征。也就是说，SkillRank 的排序逻辑并不是在复写 Star，而是在尝试从簇内内容结构中找出“被更广泛模仿、复用或靠近主流模式”的 Skill。

当然，这组案例不能被过度解读。由于当前使用的是仓库级 Star 而不是 Skill 级真值，排名靠前但 Star 很低的条目既可能是真正被低估的高质量 Skill，也可能只是出自一个较小众仓库的结构中心节点。因此，案例分析更适合用来说明 SkillRank 的行为模式，而不是直接宣称其已经发现了“绝对优质”的 Skill。

\section{讨论：当前结果的意义与局限}
\label{sec:limitations}

结合上述实验，本文认为当前结果已经足以支持两个结论。其一，SkillRank 的图结构信息确实提供了增量价值，因为调优后的配置在 100 和 1000 两个规模上都优于最强非图结构基线。其二，在当前样本量下，图结构增益是“温和但稳定”的，而不是“压倒性的领先”。

与此同时，本轮实验也清楚暴露出若干局限。

\textbf{第一，Star 只是仓库级 proxy，不是 Skill 级真值。} 同一仓库中的多个 Skill 共用同一个 Star 数，这会引入标签噪声，并削弱 within-cluster 排序评测的分辨率。

\textbf{第二，100 与 1000 规模不足以证明 FAISS 的时间优势。} 当前实验只能说明近似检索几乎不损失排序质量，但不能据此声称“FAISS 已经更快”。如果后续需要支撑大规模效率结论，必须补跑 10000 及以上规模。

\textbf{第三，当前评测仍停留在离线排序层。} 本文尚未在真实 Agent 任务中比较“注入不同 Top-$K$ Skill 集合后最终任务完成质量”的差异，因此还不能把离线 $\rho$ 的提升直接等同于真实任务成功率的提升。

\textbf{第四，聚类粒度会影响结论的强弱。} 更细的聚类有时会得到更高的 $\rho$，但往往伴随过小簇，导致评测不稳定。因此，后续研究需要引入更系统的聚类稳健性分析，而不是只选择单个最优点。

\section{全文总结}
\label{sec:conclusion}

\subsection{当期工作总结}

本文围绕 Agent Skill 生态中的价值发现问题，完成了从理论建模、算法实现到系统原型和实验评测的完整闭环。

\textbf{在理论层面}，本文做出了三项具有独立价值的贡献。其一，将 Skill 形式化为 $\langle Description, Content \rangle$ 二元结构，建立了覆盖技术与非技术全域 Skill 形态的泛化定义，厘清了需求定位与内容质量这两个相互独立的问题维度。其二，提出了条件排序问题定义：Skill 价值评估只能在语义邻域内部进行，跨邻域的全局混排在逻辑上没有意义，这一定义纠正了现有平台的根本设计错误。其三，明确阐释了 Agent 消费 Skill 与人类阅读 Page 的本质差异——Agent 的优化目标是注入集合的整体质量而非单点 Top-1 准确率，这一洞察直接决定了 SkillRank 评分设计为集合层面的质量信号而非精确排名信号。

\textbf{在算法层面}，本文实现了融合 Content 原型中心度与 PageRank 图传播分的 SkillRank 算法，并通过 AutoResearch 风格的系统调优框架对关键超参数进行迭代优化。在 100 和 1000 两个规模上，调优后的 SkillRank 分别达到 within-cluster Spearman $\rho = 0.0600$ 和 $\rho = 0.0562$，均优于最强非图结构基线，验证了图结构信息在 Skill 质量评估中的增量价值。效率实验同时证明，FAISS 近似检索在当前规模下几乎不损失排序质量（$|\Delta\rho| \leq 0.0011$），理论可扩展性框架（$O(n\log n)$）已在结构层面确立，为未来大规模部署奠定基础。

\textbf{在工程层面}，本文构建了 Taste 系统原型，打通了 GitHub 数据采集（10028 条 Skill）、离线 SkillRank 计算、在线语义路由查询、三维语义地形图可视化的完整链路，并交付了可直接运行的系统原型、完整实验日志、汇总 JSON、截图与 3D Demo 截图等全套可复现产物。

\subsection{长期价值}

本文的工作意义不仅限于当期的算法贡献，更在于它指向了一个即将到来的重要历史节点。

从宏观趋势看，Skill 生态正在经历与 2000 年代初互联网高度相似的结构性演变：数量从数千增长到数万，仍在以指数速度向十万、百万规模迈进；覆盖范围从纯技术领域向设计、电商、法律、创业等所有人类活动领域全面溢出；消费主体从少数技术用户向全量 Agent 用户扩散。在这一趋势下，”人工筛选 Skill”的模式注定走向失效——不是因为用户能力不足，而是因为规模增长的速度从根本上超过了人类认知处理的速度。

从信息论的视角看，Skill 对 Agent 的价值具有独特的结构性：能引导 Agent 采取具体行动的信息，天然比泛知识信息具有更高的信息密度，因为它消除了 LLM 作为下一词元预测系统的行动不确定性。随着 Skill 生态向非技术领域扩展，这种”人类经验的再蒸馏”将覆盖越来越多的高价值决策场景——每一个领域专家的判断力，都有可能被封装为可被 Agent 直接执行的 Skill。在这个意义上，Skill 生态最终可能演变为人类集体知识的可执行化表达。

本文提出的 SkillRank 框架，为这一趋势下的 Skill 质量自动评估提供了第一个系统性方案。正如 PageRank 在互联网黎明期建立了无需人工标注的网页质量排序体系，SkillRank 试图为 Skill 世界建立同等性质的基础设施：依靠生态自身产生的结构信号（Content 语义亲缘关系）而非人工标注，实现对 Skill 内在质量的客观、可扩展的评估。

\subsection{后续工作展望}

当前工作已完成了理论框架的建立和原型系统的验证，但在三个方向上还有重要工作有待推进。

\textbf{第一，大规模效率实验的实证闭环。}当前 100 和 1000 两个规模的效率实验尚不足以展示 FAISS 的时间优势；补充 10000 及以上规模的实验，是将”理论可扩展”转化为”实证可扩展”的必要步骤。

\textbf{第二，在线 Agent 任务层面的端到端评测。}当前评测停留在离线排序层，以 Spearman $\rho$ 衡量排序质量；更根本的验证应当是：将不同 Top-$K$ Skill 集合注入真实 Agent 任务，直接比较最终任务完成质量的差异。只有完成这一在线评测闭环，才能将离线排序质量的提升与真实 Agent 能力提升建立起直接的因果联系。

\textbf{第三，更细粒度的 Skill 级监督信号构建。}当前实验以 GitHub 仓库 Star 数作为弱监督 proxy，存在仓库级与 Skill 级的粒度不匹配问题。未来可以考虑通过 Agent 真实任务执行日志、用户反馈、或专家标注少量高置信样本等方式，构建更精确的 Skill 级质量标签，从而支撑更严格的监督评测。

只有将上述三个方向逐一补齐，SkillRank 才能从当前的”可工作的研究原型”进一步走向”可广泛部署的 Agent Skill 发现基础设施”——为即将到来的 Skill 规模爆炸提供真正意义上的系统性解法，实现本文在绪论中所展望的：为 Agent Skill 世界建立属于这个时代的 PageRank。
